纪念GitHub博客搭建成功

Mar 28, 2017 | Hits

动起手来还是很顺利的。又有自己的博客了。当初使用WordPress在阿里云搭建的博客因为半年免费期结束而被迫关闭。希望伟大的GFW不要把GitHub也墙了。本文极简单的记录一下搭建过程。

记录一下搭建过程

搭建本博客的主要工具

  • GitHub账号
  • Git for Windows软件
  • node.js
  • Hexo 静态博客框架，另有Jekyll可选
  • Atom推荐使用的编辑器,开源、免费就是启动速度太慢。sublime text非常方便就是需要$70,否则免费版会一直弹出提示。

零基础用Hexo+Github 搭建博客

博客搭建过程可能出现的问题处理

npm安装Hexo失败

国内网络原因，访问外国服务器不稳定，可以通过更换淘宝源解决，在命令行使用如下命令即可：

博客上传GitHub服务器失败，提示路径错误

deploy属性要正确设置，如果使用的GitHub Page作为博客地址， repository设置为：

deploy:
    type: git
    repository: https://github.com/your_github_username/your_github_username.github.io.git
    branch: master

如果是使用在GitHub创建的项目作为博客地址，repository地址可以登录自己的GitHub账号，进入项目复制地址，branch填写你项目的branch：

deploy:
    type: git
    repository: https://github.com/your_github_username/your_project_name.git
    branch: gh-pages

博客上传GitHub后，CSS文件丢失

默认的_config.yml文件，上传到博客根目录而不是项目目录里。需要修改一下_config.yml文件，其中的url和root属性。root 下面应该设置成 /your_project_name 而不是/ ，修改后重新部署一下就可以了。

url: http://your_github_username.github.io/your_project_name.git
    root: /your_project_name

安装博客主题时，执行npm install hexo-renderer-sass –save报如下错误：

MSBUILD : error MSB3428: 未能加载 Visual C++ 组件“VCBuild.exe”。要解决此问题，1) 安装 .NET Framework 2.0 >SDK；2) 安装 Microsoft Vis
ual Studio 2005；或 3) 如果将该组件安装到了其他位置，请将其位置添加到系统路径中。 >[D:\hexo\node_modules\node-sass\build\binding.sln]
已完成生成项目“D:\hexo\node_modules\node-sass\build\binding.sln”(默认目标)的操作 - 失败。

解决方案，最方便的解决方案是，用git shell执行本命令，先切换到Hexo目录下再执行。报上面错误肯定是windows cmd中执行报错的。解决起来还得安装vs2010 vs2012非常麻烦，不如在Windows的git shell下安装。
