简介

通常web-based app无法访问Android的硬件设备或者调用平台本身的接口，使用Cordova plugin 可以让让web-based app方便的实现上述功能。事实上Cordova的主要API均通过Plugin实现。我们使用这些API时只需安装相应的插件。

cordove plugin 可以看做由2部分组成，其一：对外统一的JavaScript接口，用户利用plugin提供的接口调用具体实现即第二部分： 其二，不同平台针对JavaScript接口的本地代码实现。

这个机制是通过调用cordova.exec() 来实现的。cordova.exec()共有五个参数依次为：

  1. 成功后的回调函数；callback
  2. 失败后的回调函数；function(err) {}
  3. Android本地方法的类名；Echo
  4. Android本地方法的方法名；echo
  5. 传递给Android本地方法的参数数组；[str]

而echo 函数本身有2个参数，function(str,callback){…} 。这里的str参数，传递给cordova.exec(),成为了第五个参数[str]。这里的callback，传递给cordova.exec()成为第一个参数，当执行完Android本地方法成功后callback被调用。

插件的生命周期和初始化

Cordova plugin对象的生命周期随WebView绑定在一起。当js调用代码被执行的时候，plugin对象被实例化（instantiated);如果希望plugin对象一开始就被创建，可以设置onload参数为true.

在plugin的本地Java类中，使用initialize方法进行初始化工作：

同样，通过覆盖onResume()、onDestroy()等方法，可以让插件实现相应生命周期的业务逻辑。当插件执行长时间的运行的请求、后台运行的activity时，比如播放媒体、listeners、internal state等，应该重写onReset()方法。

插件的线程

插件并没有运行在UI线程上，而是运行在WebCore线程。如果你需要与UI进行交互，需要使用Activity的runOnUiThread方法。
