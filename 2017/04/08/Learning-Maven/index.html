Learning Maven

Apr 8, 2017 | Hits

Maven的主要作用体现在：1、帮助我们自动化构建项目（building）。2、帮助我们管理项目引用的jar包。

Maven入门–使用Maven构建项目

安装Maven

安装过程没啥好说的。安装完后找到Maven的安装目录（以下称为M2_HOME）然后找到\conf\settings.xml文件设置一下本地仓库地址：

如果不设置这个，默认地址在C:盘。放在系统盘有两个不好的地方：

  • 系统盘空间往往有限，很多软件的默认地址都是存放系统盘
  • 格式化重装系统后系统盘内容全部丢失，有些时候重装系统还不得不格式化

安装完Maven可以参考官方getting started去开始使用并熟悉Maven。

Maven常见用法

简单介绍Maven命令格式：

options 是一些选项，比如 -e 表示显示详细错误，可以通过mvn -h查看options的详细介绍。

goal 表示需要mvn去完成的一个任务:

mvn package 表示打包而mvn deploy表示部署，可以在一个mvn命令中指定多个任务用空格分割，这在使用clean时很常见，比如mvn clean install 表示先执行清理再执行安装。goal的完整写法应该是plugin:goal ,比如:deploy的完整形式应该是deploy:deploy.
如果需要了解goal可以查看Plugin文档,点击plugin的链接，查看Goals Overview即可.

常见Maven命令

主要参考了一下这里,写这篇文字的时候百度出来的。
mvn test 运行程序中的单元测试
mvn compile 编译项目
mvn package 打包，此时target目录下会出现jar文件
mvn install 打包并安装到本地仓库

mvn archetype:generate 创建maven项目
mvn package 打包，
mvn package -Prelease打包，并生成部署用的包，比如deploy/*.tgz
mvn install 打包并安装到本地库
mvn eclipse:eclipse 生成eclipse项目文件
mvn eclipse:clean 清除eclipse项目文件
mvn site 生成项目相关信息的网站

mvn -Dsurefire.useFile=false如果执行单元测试出错，用该命令可以在console输出失败的单元测试及相关信息
set MAVEN_OPTS=-Xmx512m -XX:MaxPermSize=256m 调大jvm内存和持久代，maven/jvm out of memory error
mvn -X maven log level设定为debug在运行
mvndebug 运行jpda允许remote debug
